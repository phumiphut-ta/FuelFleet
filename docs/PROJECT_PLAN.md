# แผนการดำเนินงานและโครงสร้างภาระงาน (Work Breakdown Structure: WBS)
## โครงการพัฒนาแอปพลิเคชันบริหารจัดการรถส่วนกลางและควบคุมโควต้าน้ำมัน (FuelFleet™)
**ระยะเวลาดำเนินงาน:** 15 กรกฎาคม 2569 – 30 กันยายน 2569 (รวมระยะเวลา 11 สัปดาห์)  
**วัตถุประสงค์เอกสาร:** เพื่อใช้ประกอบการขออนุมัติปฏิบัติงานล่วงเวลา (Overtime) และการปฏิบัติงานในวันหยุด (Weekend Work)

## 🛠️ รายละเอียดภาระงานรายสัปดาห์และการปฏิบัติงานล่วงเวลา (Weekly Task Breakdown)

### 📌 เฟส 1: การวางโครงสร้างและออกแบบฐานข้อมูล (15 ก.ค. – 25 ก.ค. 2569)
*เน้นการวางฐานรากของระบบเชิงสถาปัตยกรรมและการสร้างโมเดลความสัมพันธ์ของข้อมูล (ER Diagram)*

* **สัปดาห์ที่ 1 (15 - 19 ก.ค. 2569):**
  * รวบรวมขอบเขตและเงื่อนไขทางธุรกิจ (Business Rules) ของโควต้าน้ำมันรายคันและการจองรถข้ามจังหวัด
  * ออกแบบ Schema ฐานข้อมูลเชิงสัมพันธ์ MySQL (ตาราง Employee, Car, Booking, Quota, Receipts, Audit Log)
  * จัดตั้งสภาพแวดล้อมการพัฒนา (Setup Laravel/PHP Composer environment, PSR-4 Autoloading)
  * **[ขออนุมัติทำงานวันหยุด 18-19 ก.ค. 69]:** เพื่อทำการปรับแต่งดัชนี (Indexes) และทดสอบความเข้ากันได้ของคีย์นอก (Foreign Keys/Cascades) ในฐานข้อมูล จำเป็นต้องใช้เวลาต่อเนื่องเพื่อไม่ให้สตรีมของงานออกแบบฐานข้อมูลล่าช้า
* **สัปดาห์ที่ 2 (20 - 25 ก.ค. 2569):**
  * เขียนไฟล์ติดตั้งฐานข้อมูลแรกเริ่ม (`schema.sql` และระบบ Seeding ข้อมูลจำลองพนักงาน/แผนก/รถยนต์)
  * พัฒนาคลาสเชื่อมต่อฐานข้อมูลแกนกลาง (`App\Core\Database`) แบบ Singleton pattern เพื่อความเสถียร
  * **[ขออนุมัติทำงานล่วงเวลา (OT) 21, 23 ก.ค. 69]:** ทดสอบจำลองเชื่อมต่อกับเซิร์ฟเวอร์ MySQL บนเครื่อง Windows Server/IIS เพื่อตรวจสอบปัญหาการเข้ารหัสตัวอักษรภาษาไทย (Collation UTF-8)

### 📌 เฟส 2: การพัฒนาส่วนประมวลผลหลัก (Core Engine & Booking) (26 ก.ค. – 16 ส.ค. 2569)
*การสร้างเครื่องมือตรวจสอบเงื่อนไข (Engine) ทั้งระบบความปลอดภัย สิทธิ์ และตรรกะการจองรถไม่ให้ทับซ้อนกัน*

* **สัปดาห์ที่ 3 (26 - 31 ก.ค. 2569):**
  * พัฒนาระบบตรวจสอบสิทธิ์และเซสชัน (Bcrypt Password Hashing Admin Login/Logout)
  * พัฒนาส่วนจัดการข้อมูลรถยนต์หลวง (ตารางและฟอร์ม CRUD ยานพาหนะ, สถานะ Active/Suspended)
  * พัฒนาระบบปฏิทินจองรถพื้นฐาน ดึงข้อมูลการจองด้วย AJAX
  * **[ขออนุมัติทำงานวันหยุด 25-26 ก.ค. 69]:** ดำเนินการพัฒนาระบบตรวจสอบเวลาทับซ้อน (Booking Overlap Validation Engine) ซึ่งเป็นหัวใจสำคัญ ป้องกันไม่ให้รถคันเดียวกันโดนจองในวันเวลาเดียวกัน
* **สัปดาห์ที่ 4 (1 - 9 ส.ค. 2569):**
  * พัฒนาตรรกะการคำนวณโควต้าน้ำมันรายรถ (Quota Calculation Engine) ในแต่ละช่วงเดือน
  * พัฒนาระบบระงับการจองใช้งานรถยนต์หลวงชั่วคราว (Car Suspension Management) สำหรับกรณีรถซ่อมบำรุง
  * **[ขออนุมัติทำงานล่วงเวลา (OT) 3, 5, 7 ส.ค. 69]:** ปรับแต่งระบบ Routing ของเฟรมเวิร์ก เพื่อให้รองรับการติดตั้งภายใต้ไดเรกทอรีย่อยบน IIS Server (IIS Sub-Application)
* **สัปดาห์ที่ 5 (10 - 16 ส.ค. 2569):**
  * พัฒนาระบบบันทึกประวัติการกระทำของผู้ใช้หลังบ้าน (Audit Logging System) เพื่อเก็บประวัติล็อกอิน แก้ไขโควต้า และยกเลิกใบเสร็จ
  * พัฒนาระบบจัดการข้อตกลงและเงื่อนไขการจองรถ (Booking Agreements CRUD & Sort Order)
  * **[ขออนุมัติทำงานวันหยุด 15-16 ส.ค. 69]:** ทำการพัฒนาระบบสับเปลี่ยนลำดับข้อตกลงการจอง (Agreement Reordering UI/UX ด้วย Drag & Drop/Up-Down) และทำประวัติ Audit Logs ควบคู่กัน

### 📌 เฟส 3: พัฒนา UI คอนโซล, หน้าแนบหลักฐาน และ LINE LIFF (17 ส.ค. – 6 ก.ย. 2569)
*พัฒนารูปลักษณ์หน้าจอ Luxury Dark Mode, ระบบรองรับเอกสาร PDF/ภาพถ่าย และการเข้าถึงทางมือถือ*

* **สัปดาห์ที่ 6 (17 - 23 ส.ค. 2569):**
  * ออกแบบและพัฒนากรอบเลย์เอาต์หลักหลังบ้าน (Admin Sidebar & Responsive Grid Layout) ด้วยสไตล์ Glassmorphism
  * พัฒนาระบบความเสถียรของฟอร์มส่งข้อมูล ให้รองรับการอัปโหลดไฟล์ขนาดใหญ่แบบไม่ประสานเวลา (Asynchronous File Upload Check)
  * **[ขออนุมัติทำงานวันหยุด 22-23 ส.ค. 69]:** ออกแบบ Dropzone สำหรับแนบไฟล์ใบเสร็จ (Drag & Drop Zone) รองรับทั้ง PDF และรูปภาพ พร้อมระบบวิเคราะห์ความกว้าง/ความสูงของภาพและขนาดหน่วยความจำ
* **สัปดาห์ที่ 7 (24 - 30 ส.ค. 2569):**
  * พัฒนาหน้าการอนุมัติใบเสร็จน้ำมันสำหรับแอดมิน (Receipt Verification/Approval Console)
  * พัฒนาหน้าจอบันทึกประวัติใบเสร็จย้อนหลัง (Historical Data Import Console)
  * **[ขออนุมัติทำงานล่วงเวลา (OT) 25, 27, 29 ส.ค. 69]:** ออกแบบหน้ารวมสถิติล่าสุดสาธารณะ (Recent Receipts View) ปรับแต่งการคัดกรองข้อมูลด้วยฟังก์ชัน SQL `LEFT JOIN` พนักงานและตำแหน่ง เพื่อแก้ไข Warning ต่างๆ
* **สัปดาห์ที่ 8 (31 ส.ค. – 6 ก.ย. 2569):**
  * พัฒนาหน้าจอแสดงโควต้าน้ำมันรายเดือนสำหรับผู้ใช้ผ่านทางอุปกรณ์พกพา (Mobile-first LINE LIFF View)
  * ติดตั้งหลอดพลังงานแสดงสัดส่วนการใช้น้ำมัน (Draining bar progress 100% -> 0% และแถบสีเตือนกรณีใช้เกินโควต้า)
  * **[ขออนุมัติทำงานวันหยุด 5-6 ก.ย. 69]:** ปรับแต่งระบบปัดหน้าจอแนวนอนบนมือถือ (Horizontal Swipe UI) สำหรับแถบเมนูใน LINE LIFF และการปรับแต่งปฏิทิน FullCalendar ให้แสดงผลแบบ Responsive บนหน้าจอ iPhone/iPad

### 📌 เฟส 5: การทดสอบระบบความปลอดภัยและประสิทธิภาพความทนทาน (21 ก.ย. – 27 ก.ย. 2569)
*การันตีเสถียรภาพการทำงาน ปราศจากบั๊ก และป้องกันการถูกเจาะระบบ*

## 💡 เหตุผลและความจำเป็นในการปฏิบัติงานวันหยุดและวันหยุดนักขัตฤกษ์ (Justification for Holiday Work)
1. **ความต่อเนื่องของตรรกะระบบ (Logical Continuity):** 
   โมดูลประมวลผลหลัก เช่น เครื่องคำนวณการทับซ้อนเวลาจองรถ (Booking Overlap Engine) และระบบความปลอดภัย (CSRF / SQL Injection Hardening) เป็นโมดูลที่มีความเชื่อมโยงสูงและซับซ้อน การหยุดดำเนินงานข้ามวันอาจทำให้เกิดช่องโหว่ทางตรรกะ การพัฒนาอย่างต่อเนื่องในวันหยุดทำให้การเขียนตรรกะเสร็จสมบูรณ์รวดเร็วและปราศจากข้อผิดพลาด
2. **การทำเคสทดสอบและการจำลองโหลด (Testing Isolation):** 
   การรันชุดทดสอบความทนทานและการเจาะระบบ (Penetration & Performance testing) จำเป็นต้องใช้สภาพแวดล้อมที่ไม่มีผู้ใช้งานภายนอกมารบกวน เพื่อให้ค่าสถิติการตอบสนองคงที่และไม่ไปขัดขวางการทำงานของระบบเครือข่ายภายในอื่นๆ
3. **การเข้าถึงระบบในช่วงเวลาพักเครื่อง (Maintenance Window Deployments):** 
   การตั้งค่าพอร์ตเซิร์ฟเวอร์จริง (Deployment/Database Alteration) ต้องทำนอกเวลาทำการและช่วงวันหยุดเสาร์-อาทิตย์ เพื่อไม่ให้กระทบต่อพนักงานที่จำเป็นต้องเข้าดูสถิติจองรถยนต์หลวงหรือใช้งานระบบอื่นๆ ประจำวัน
